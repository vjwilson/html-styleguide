HTML Styleguide

This is an example of an HTML styleguide

This styleguide was inspired by Isobar Front-end Code Standards.

Headings

Headings should be used to label the hierarchical sections of a document, not to display text in larger font sizes.

Examples of headings

<h1>

Top-level heading.

Every document must have one, and should only have one <h1> element. (In cases where there are multiple documents included on one physical webpage, for instance, in a list of blog posts, some people recommend that the title of each document in the list have an <h1> element for its title, because each document could stand on its own.)

<h2>

Second-level heading

Titles of sections within a document should be <h2> elements. These can be sub-sections of an article, or ancillary sections of a page, such as a sidebar with a title of "Related Articles" that lists other documents related to the main document of the page.

<h3>

Third-level heading

Titles of sections within any second-level sections should be <h3> elements. These can be sub-sections of an article, or parts of a ancillary sections of a page, for instance, the sections "Related Articles", "Latest Tweets", and "Categories" in a blog sidebar.

<h4>

Fourth-level heading

Titles of sections within any third-level sections should be <h4> elements.

<h5>
Fifth-level heading

Titles of sections within any fourth-level sections should be <h5> elements.

<h6>
Sixth-level heading

Titles of sections within any fifth-level sections should be <h5> elements.

Paragraphs

Body copy should be wrapped in <p> tags. Breaks should be inserted by closing one <p> element and opening another.

Do not use <br> elements to insert line breaks in copy, except in cases where it is the simplest solution to display units of related information that normally appear on seperate lines, such as postal addresses.

Examples of paragraphs

<p>

This is one idea.

This is another idea.

<br>

This is a physical address:
Trinity Church
75 North Broadway
New York, NY 10006
